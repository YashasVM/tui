# tui-portfolio

A real terminal portfolio with a Retro UNIX shell interface. It is built for developer audiences: fast scanning, GitHub-first project data, keyboard-driven exploration, and editable local JSON content.

## Quick Start

```bash
npm install
npm run dev
```

Build and run the local package entry:

```bash
npm run build
npx .
```

Sync public GitHub repositories into the local cache:

```bash
npm run sync:github -- --user your-github-username
```

or after building:

```bash
npx . sync --user your-github-username
```

## Shell Commands

Inside the portfolio shell:

```text
help
about
projects
open <id|repo-name|number>
skills
github
contact
resume
theme amber|green|paper
clear
exit
```

## Content

Edit these files to make the portfolio yours:

- `data/profile.json`: name, title, bio, links, resume, skills, and experience.
- `data/projects.json`: curated projects and highlights.
- `data/github.generated.json`: generated by the GitHub sync command.

Curated project fields in `data/projects.json` override matching GitHub fields. Match a GitHub repository with the `repo` field, for example:

```json
{
  "id": "my-tool",
  "repo": "your-github-username/my-tool",
  "name": "my-tool",
  "description": "A short project description.",
  "stack": ["TypeScript", "Node.js"],
  "highlights": ["What makes this project worth opening"],
  "featured": true
}
```

## Development

```bash
npm run build
npm test
```

The CLI uses local JSON by default from `./data`. To point it elsewhere:

```bash
tui-portfolio --data-dir ./path/to/data
```
